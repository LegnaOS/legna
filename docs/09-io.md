## 9.1 output

将内容写入标准输出。支持字符串字面量、整数变量、字符串变量和表达式：

```legna
output "hello\n"       # 字符串字面量
output x               # 整数变量（自动转为十进制文本）
output name            # 字符串变量
output x + 1           # 表达式
```

**缓冲机制：** 所有输出写入 4KB 运行时缓冲区，程序退出时统一 flush。这大幅减少系统调用次数，提升 I/O 密集型程序的性能。

## 9.2 input_num()

从标准输入读取一行，解析为整数：

```legna
let x = input_num()
output x * 2
```

## 9.3 input_str()

从标准输入读取一行，存为字符串：

```legna
let name = input_str()
output "hello, "
output name
output "\n"
```

## 9.4 emit（结构化输出）

`emit` 输出 JSON Lines 格式到标准输出，每次调用产生一行合法的 JSON 对象：

```legna
emit "status" "ok"       # → {"status":"ok"}
emit "count" 42          # → {"count":42}
emit "total" x + y       # → {"total":30}
```

**格式规则：**

- key 必须是字符串字面量
- value 为整数时不加引号：`{"key":42}`
- value 为字符串时加引号：`{"key":"value"}`
- 字符串 value 中的 `"` 和 `\` 自动 JSON 转义
- 每行一个完整 JSON 对象，以 `\n` 结尾

`emit` 会先 flush 输出缓冲区再写入，确保 JSON 行不与 `output` 内容混合。

## 9.5 文件 I/O

### open

打开文件，返回文件描述符（整数）：

```legna
let fd = open("data.txt", "r")    # 只读模式
let out = open("result.txt", "w") # 写入模式（创建或截断）
```

模式参数：`"r"` 只读，`"w"` 写入（不存在则创建，存在则截断）。

### read_line

从文件描述符读取一行，返回字符串：

```legna
let line = read_line(fd)
output line
```

读取到文件末尾时返回空字符串。

### write_line

向文件描述符写入数据：

```legna
write_line fd "hello\n"    # 写入字符串
write_line fd 42           # 写入整数（自动转为文本）
```

### close

关闭文件描述符：

```legna
close(fd)
```

### 完整示例

```legna
legna:
    # 写入文件
    let out = open("output.txt", "w")
    write_line out "hello, world\n"
    close(out)

    # 读取文件
    let fd = open("output.txt", "r")
    let content = read_line(fd)
    output content
    close(fd)
```

## 9.6 I/O 格式规范

Legna 的结构化 I/O 采用 [JSON Lines](https://jsonlines.org/) 格式：

- 每行一个合法的 JSON 对象
- 对象只有一个键值对：`{"key":value}`
- key 始终是字符串
- value 是整数或字符串
- 行以 `\n` 分隔

这种格式天然适配 AI agent 工作流：

```bash
# 用 jq 解析 Legna 程序输出
./my_program | jq -r '.status'

# 用 Python 解析
import json, subprocess
out = subprocess.check_output(["./my_program"])
for line in out.strip().split(b"\n"):
    data = json.loads(line)
    print(data)
```
